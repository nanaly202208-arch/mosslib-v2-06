# 情报罗盘模块 — 功能清单（修正版）

> **说明**：本表基于原型实现与 PRD 文档整理，供开发与测试人员对照实施。红色标注为与上一版清单相比修正或补充的内容。

## 一、类型-颜色映射表

| 类型 | CSS 类名 | 文字颜色 | 悬停背景 | 说明 |
|------|----------|----------|----------|------|
| 武器装备 | `entity-weapon` | `#B91C1C` | `rgba(185,28,28,0.06)` | 武器、无人机、导弹等 |
| 企业 | `entity-company` | `#1E40AF` | `rgba(30,64,175,0.06)` | 公司、厂商、集团 |
| 机构/组织 | `entity-org` | `#6B21A8` | `rgba(107,33,168,0.06)` | 政府、军队、国际组织 |
| 人物 | `entity-person` | `#9A3412` | `rgba(154,52,18,0.06)` | 个人姓名 |
| 技术/概念 | `entity-tech` | `#0F766E` | `rgba(15,118,110,0.06)` | 技术术语、协议、标准 |

> 颜色方案为文字色 + 底部虚线边框，悬停时加粗边框 + 浅色背景，无需背景填充色块。

## 二、原文实体高亮

| 功能点 | 详细说明 | 交互 / 联动 / 异常处理 | 自测验收标准 |
|--------|----------|------------------------|--------------|
| **数据标注** | 原文段落中的实体通过 `<span class="entity-{type}">` 内联标记，type 取值为 weapon / company / org / person / tech | 标注位置由后端在 `originalText` 中插入 `<span>` 实现；若后端未标注则原文正常显示 | 原文中所有实体均有对应类型标注，位置准确 |
| **颜色渲染** | 按类型-颜色映射表渲染文字颜色 + 底部虚线边框。悬停时边框加粗 + 浅色背景 | 鼠标移入：border-bottom-width 1px→2px，添加浅色背景；鼠标移出还原 | 颜色与类型一一对应，文字清晰可读，悬停反馈明显 |
| **点击行为** | 点击高亮文字 → 打开实体详情抽屉；同时右侧面板对应卡片置为选中态 | 有详情数据的实体打开抽屉；无详情数据的实体弹出"暂无数据"Toast | 点击高亮文字，抽屉正确打开；无数据时 Toast 提示 |
| **悬浮卡片（Hover Card）** | 鼠标悬停高亮文字 300ms 后，弹出预览卡片 | 卡片内容：实体名称（原文+中文）、类型标签、简介摘要、缩略图；卡片跟随鼠标位置弹出；鼠标移入卡片区域时保持显示；移出实体文字或卡片 500ms 后消失 | 悬停延迟合理，卡片不闪烁，内容正确；可移动鼠标到卡片上点击 |

## 三、右侧实体面板

| 功能点 | 详细说明 | 交互 / 联动 / 异常处理 | 自测验收标准 |
|--------|----------|------------------------|--------------|
| **面板布局** | 固定于右侧，宽度 460px，内容超出时内部滚动（不撑开页面） | 面板高度填满可视区；scroll 容器为 `.right-panel-scroll` | 面板宽度固定 460px，内部可滚动，不溢出页面 |
| **分组展示** | 实体按类型分组：武器装备、企业、组织/机构、人物、技术/概念。组内按首次出现顺序排序 | 各组可折叠/展开（默认折叠）；折叠状态本地持久化 | 分组正确，组内无重复实体，排序按首次出现 |
| **类型筛选标签** | 面板顶部显示类型标签行：武器装备、企业、组织/机构、人物、技术/概念 | 点击标签滚动到对应分组（`scrollIntoView`）；选中标签加下划线高亮 | 点击标签准确定位到对应分组，标签高亮正确 |
| **卡片展示** | 每张卡片包含：实体名称（原文）、中文名/别名、类型标签（颜色标识） | 卡片布局：左边图标头像，右侧名称 + 中文名 + 类型标签 | 卡片内容完整，类型标签颜色正确 |
| **卡片状态** | 支持普通态、hover 态、选中态 | hover：背景加深、鼠标变手型、边框高亮；选中态：左侧加粗指示条，由点击或文章联动触发 | 三种视觉状态区分明显 |
| **卡片点击** | 点击卡片 → 打开实体详情抽屉；卡片保持选中态 | 有详情数据的实体打开抽屉；无详情数据的实体弹出"暂无数据"Toast；**武器装备分组第一个 Orion 特殊处理：仅高亮文章中的 Orion，不打开抽屉** | 点击卡片抽屉打开；Orion 卡片不打开抽屉，仅高亮文章 |
| **悬停联动（文章→面板）** | 鼠标悬停文章内高亮词 → 右侧面板对应卡片边框高亮，自动滚动至该卡片可见区域 | 使用 `scrollIntoView`，需处理面板内滚动偏移；移出时恢复高亮，保留最后一次点击的选中态 | 悬停联动准确，卡片滚入视野，无突兀跳动 |

## 四、实体详情抽屉

| 功能点 | 详细说明 | 交互 / 联动 / 异常处理 | 自测验收标准 |
|--------|----------|------------------------|--------------|
| **抽屉打开/关闭** | 右侧滑出，宽度 580px，动画 300ms ease-out | 遮罩黑色 50% 透明度；支持关闭按钮与点击遮罩关闭；页面 body 禁止滚动 | 动画流畅，遮罩正确，body 不滚动 |
| **图片轮播区** | 顶部轮播图，4s 自动切换，支持手动翻页（左右箭头 + 底部导航点） | 自动切换循环播放；手动点击箭头或导航点切换；无图片时不显示轮播区域 | 轮播自动/手动切换正常，图片加载正确 |
| **头部区域** | 依次展示：标签行 → 实体名称（原文 28px + 中文 16px）→ 承包商/别名（带图标） | 标签行展示在标题上方；承包商和别名前有蓝灰色图标 | 头部布局正确，字号、颜色符合规范 |
| **简介区域** | 富文本概述实体背景信息 | 支持 `<strong>` 等 HTML 标签渲染 | 简介内容完整，富文本正确渲染 |
| **KPI 卡片网格** | 2×2 网格展示核心战技指标 | 卡片 hover 时边框高亮 + 上浮阴影；标签 12px，数值 14px font-weight:800 | 网格布局正确，hover 动效正常 |
| **技术规格表** | 2 列表格展示技术参数（标签12px / 值14px） | 无额外交互 | 表格内容完整 |
| **挂载装备** | 2 列卡片展示可挂载武器装备 | 卡片显示：类型标签、装备名称、说明、挂载数量 | 卡片布局正确，信息完整 |
| **主要使用方** | 展示使用方国旗/名称/数量/部署情况 | 无额外交互 | 信息展示完整 |
| **动态追踪时间轴** | 展示实体关键事件时间线 | 左侧竖线时间轴 + 圆点节点；hover 时节点放大 + 阴影；事件按时间倒序排列 | 时间轴布局正确，hover 动效正常 |
| **内容切换** | 在抽屉打开状态下点击另一实体，直接替换内容，不叠加多层 | 切换时轮播索引重置并重新启动自动播放 | 无抽屉叠加，内容替换无闪烁 |

## 六、暂无数据 Toast

| 功能点 | 详细说明 | 验收标准 |
|--------|----------|----------|
| **触发场景** | 点击无详情数据的实体卡片/文章实体/悬浮卡片时触发 | 仅对无 `entityDetails` 数据的实体生效 |
| **展示效果** | 页面居中弹出半透明深色提示"暂无数据" | 提示文案正确，2s 后自动消失，不影响页面操作 |
| **异常处理** | 不阻塞页面其他交互 | 点击后 Toast 显示期间，页面其他功能正常 |

## 七、异常与边界

| 功能点 | 详细说明 | 验收标准 |
|--------|----------|----------|
| **无实体段落** | 段落无任何实体时正常显示文字，右侧面板不出现对应卡片 | 不影响文章排版，面板无多余卡片 |
| **长文本性能** | 文章极长（5 万字符+）时，高亮渲染不卡顿 | 滚动、悬停、点击等交互帧率 > 30fps |
| **详情加载失败** | 详情接口失败（API 模式下）展示错误提示与重试按钮 | 点击重试可重新请求 |
| **响应式预留** | 不做移动端适配，但使用相对单位/弹性布局 | 视口缩小到平板尺寸时布局无明显崩坏 |
| **武器装备第一个 Orion** | 当前正在展示详情的实体，其卡片点击仅高亮文章中的对应位置，不打开抽屉 | Orion 卡片点击后文章高亮，抽屉不重复打开 |

## 八、数据接口（API 化后适用）

| 接口 | 说明 | 字段要求 | 异常处理 |
|------|------|----------|----------|
| `GET /api/articles/{articleId}` | 返回文章段落、原文/译文、实体标记、实体列表 | `paragraphs[]` 包含 `originalText`（原文段落带 `<span>` 标注）和 `translatedText`（译文段落）；`entityList[]` 按类型分组的去重实体列表 | 接口超时/失败展示错误 UI |
| `GET /api/entities/{entityId}` | 返回实体详情数据 | 包含：名称（原文/中文）、类型、图片数组、KPI 列表、规格列表、挂载装备、使用方、时间轴事件、简介 | 失败时抽屉内显示错误提示 + 重试按钮 |

> **说明**：当前原型阶段数据为本地硬编码（`entityDetails` 对象 + 内联 `<span>` 标注），API 接口是后续改造目标。
